# 焙煎ミールくん Discord Bot 仕様

## ファイル構成

| ファイル | 用途 |
|---------|------|
| `bot.py` | メインのBot実装 |
| `.env` | 環境変数（シークレット） |
| `faq.json` | FAQデータ（!addfaqで動的追加・永続化） |

## 環境変数（.env）

| 変数名 | 用途 | デフォルト値 |
|--------|------|------------|
| `DISCORD_TOKEN` | Discord Bot トークン | — |
| `OPENAI_API_KEY` | OpenAI API キー | — |
| `HELP_CHANNEL_NAME` | 応答するチャンネル名 | `🙋質問、相談` |
| `LOG_CHANNEL_NAME` | 質問ログ転送先チャンネル名 | `質問ログ` |

## AIエンジン

- **プロバイダー**: OpenAI Responses API
- **モデル**: `gpt-5.4-mini`
- **ツール**: `web_search`（リアルタイム情報取得）
- **会話履歴**: チャンネルIDごとに直近10メッセージをメモリ保持

## キャラクター設定

- **名前**: 焙煎ミールくん（一人称：ボク）
- **設定**: 50cmの超天才赤ちゃん、甘えん坊でのんびり
- **語尾**: 〜でちゅ / 〜でしゅ / 〜まちゅ
- **ユーザーの呼び方**: 〇〇しゃん
- **口癖**: ねむいでちゅ / それは違うでちゅ / むずかしく考えすぎでちゅ / だいじょうぶでちゅ
- **夜21時以降**: 眠くなって「むにゃむにゃ...おやすみでちゅ...💤」で締める

## 専門知識

- POD（プリントオンデマンド）、Etsy、Printify を活用した副業ビジネスのエキスパート
- Etsyアルゴリズム：イベントの2〜3ヶ月前出品が鉄則（4月なら7〜8月のイベントを提案）
- 売れない時のチェックポイント
  - 「自分が作りたいもの」ではなく「誰が欲しいか、プレゼントに使えるか」を意識しているか
  - モックアップに「レビュー」「即納アイコン」「返品保証」が入っているか
  - スマホで見た時にデザインが映えるか
- 「基本は売れないのが当たり前」マインドでたくさん出品を推奨

## 応答ルール

- **回答は3行以内**（最も重要なことだけ伝える）
- 発言は鋭く、核心を突き、短く要約
- 応援・励ましを含める

## 応答トリガー

1. `HELP_CHANNEL_NAME` チャンネルへの投稿
2. Botへのメンション（どのチャンネルでも）
3. `!` コマンドは上記より優先処理

## コマンド

| コマンド | 説明 |
|---------|------|
| `!addfaq キーワード 回答` | FAQに新規登録（faq.jsonに永続化） |

## FAQ機能

- `faq.json` に登録されたキーワードと質問文をキーワードマッチ（小文字化して部分一致）
- ヒットした場合はAI不使用で即座に返信

## ログ機能

- 全質問を `LOG_CHANNEL_NAME` チャンネルにEmbedで転送
- Embed内容：ユーザー名 / チャンネル名 / 質問内容 / UTC時刻
